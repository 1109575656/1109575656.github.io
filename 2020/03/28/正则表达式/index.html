正则表达式

正则
正则
发布日期:   2020-03-28

正则表达式：字符串匹配，查找等

1.元字符（1个字母，符号，汉字等）
^：匹配字符串的开始
$：匹配字符串的结束
.：除\n之外的任何一个字符
-：范围 0-9 、a-z，A-Z\d：数字
\d：匹配数字0-9
\D： ^\d 匹配非数字
\s：匹配任意的空白符
\S：^\s 匹配非空白符
\b： 匹配单词的开始或结束 \babc：以adc开始的 abc\b：以abc结尾的 \babc\b：以abc开始并且以abc结尾的
\B ：^\b 匹配单词的非开始或结束
\w：匹配字母或数字或下划线或汉字
\W：^\w，除字母或数字或下划线或汉字的任何字符
[0-9a-zA-Z] 匹配任意一个数字和字母
[字符&*.等] 不需要转义

2.个数，必须配合元字符/表达式/字词使用

  • 重复任意次
    ？ 0 /1个，有或者没有
  • 重复1次或更多次
    {n} 重复n次
    {n,} 重复n次或更多次
    {n,m} 重复n到m次

3.条件：
^ not
| or
&& and

4.分组/子表达式：（）
每一对()会分配一个编号，使用()的捕获根据左括号的顺序从1开始自动编号
1.通过组号引用分组
语法：\n
2.通过组名引用分组
定义：(?exp)
使用：\k
3.无法引用的分组
语法：(?:exp)
注意：不为该分组分配组号，不计入组号

5.贪婪与懒惰
贪婪匹配：尽可能多的匹配
懒惰匹配：尽可能少的匹配，用的较多
语法：次数+?
*？ 重复任意次，但尽可能少重复
+？ 重复1次或更多次，但尽可能少重复
?？ 重复0/1次，但尽可能少重复
{n,}？ 重复n次或更多次，但尽可能少重复
{n,m}？ 重复n到m次，但尽可能少重复

断言=真假/是否，零宽=不占用字符
6.零宽断言：对位置的匹配
1.零宽度正预测先行断言：(?=exp)
人话：断言此位置的后面能匹配表达式exp
精简：后边必须干啥
2.零宽度正回顾后发断言：(?<=exp)
人话：断言此位置的前面能匹配表达式exp
精简：前边必须干啥

7.负向零宽断言
1.零宽度负正预测先行断言：(?!exp)
人话：断言此位置的后面不能匹配表达式exp
精简：后边必须不能干啥
2.零宽度负正回顾后发断言：(?<!exp)
人话：断言此位置的前面不能匹配表达式exp
精简：前边必须不能干啥

8.冗长的平衡组和递归匹配
(?’group’) 把捕获的内容命名为group,并压入堆栈(Stack)
(?’-group’) 从堆栈上弹出最后压入堆栈的名为group的捕获内容，如果堆栈本来为空，则本分组的匹配失败
(?(group)yes|no) 如果堆栈上存在以名为group的捕获内容的话，继续匹配yes部分的表达式，否则继续匹配no部分
(?!) 零宽负向先行断言，由于没有后缀表达式，试图匹配总是失败

9.注释
1.(?#注释)
2.开启Free-spacing，输入#注释

c#代码：
using System.Text.RegularExpressions;
Regex re = new Regex(“正则表达式”);
MatchCollection matchs = re.Matches(“匹配字符串”);
Match match=re.Match(“匹配到的第一个结果”);
